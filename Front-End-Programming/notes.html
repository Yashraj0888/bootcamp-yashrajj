Notes: Exercises - Frontend & Vite

Objective: Build a student management dashboard using pure HTML, CSS, and JavaScript. This includes a table to display student records and a few interactive features like adding rows, changing the font dynamically, and showing browser information.

Exercise 1: Basic Frontend

Click to view Demo

Features to Implement:

  • Table Structure: A table to display student data with headers: ID, Name, Age, Grade.
  • Add Student Button: A button to add a new row to the table with dummy student data.
  • Highlight New Rows: Dynamically apply a highlight CSS class to newly added rows.
  • Change Font Button: A button that changes the font of the table using a Google Font.
  • Display Browser Info: Show browser name, version, window dimensions, and user agent below the table.

Steps Involved:

  • DOM Manipulation: Use JavaScript to dynamically add rows to the table and highlight them.
  • CSS Styling: Use inline CSS to style the table, buttons, and highlighted rows.
  • Google Font Integration: Dynamically load a Google font when a button is clicked.
  • Browser Information: Use JavaScript to fetch and display the browser name, version, and other details.

CSS Manipulation:

  • Style the table with borders, padding, and background color.
  • Apply a `highlight` class to newly added rows for emphasis (e.g., change background color).
  • Style buttons with hover effects for interactivity.

Exercise 2: Using Tools

Click to view Demo

Objective:

Learn how to package and structure projects using TypeScript. Follow a development life cycle: Development → Testing → Production.

Steps:

  • Set Up a TypeScript Project:
    • Initialize a new project using a package manager like npm.
    • Configure TypeScript using a tsconfig.json file.
  • Three Problems to Solve:
    • Problem 1: Load a JSON file, render a table dynamically, and include sorting functionality.
    • Problem 2: Fetch data from a CSV file using PapaParse. Enhance the table with column filtering and row highlighting.
    • Problem 3: Integrate a build tool (e.g., tsc for transpilation). Package your project and ensure the app works in both development and production environments.
  • Dev, Test, and Production Life Cycle: Use Vite in your package.json for:
    • Development server with live reload.
    • Linting and type-checking during development.
    • Production build with bundling.

Exercise 3: Starting with Vite

Click to view Demo

Objective:

Learn how to use Vite to scaffold, configure, and build projects with modern tools.

Steps:

  • Starter Template (JavaScript): Initialize a Vite project using the JavaScript template: 
    npm create vite@latest my-js-app -- --template vanilla
  • Task: Rebuild the student management dashboard from Exercise 1 using Vite, load student data from a JSON file, and display it in a table.
  • Starter Template (TypeScript): Initialize a Vite project using the TypeScript template: 
    npm create vite@latest my-ts-app -- --template vanilla-ts
  • Task: Enhance the student management dashboard by fetching data from a CSV file, parse it using PapaParse, and display it using AG Grid. Modularize the code for data loading, parsing, and rendering.
  • Vite with Extended Options: Define a CSS file with classes like m-1, m-2 until m-8. Each m-n is n*2px. Now do the same in SCSS, with loops. See how you can do it with Vite (it supports SCSS).

Exercise 4: Vite Full Exercise

Objective:

Use Vite to its full potential, implementing advanced features and best practices.

Steps:

  • Environment-Specific Configurations: Set up .env.development and .env.production files to define separate API endpoints.
  • Task: Create an app that fetches data from an endpoint specified in the environment variables. Ensure the correct environment file is used in development (npm run dev) and production (npm run build).
  • Advanced Vite Practices: Use vite-plugin-checker to integrate real-time TypeScript type-checking and vite-plugin-eslint for linting in the development workflow. Add linting and type-checking tools to your Vite TypeScript project and display errors during the build process.
  • Build Upon Previous Work: Add advanced features like lazy loading modules, code splitting, optimize for production by ensuring minification and tree-shaking, and test the performance of the final build.
