Exercise 4: Debugging with Debug Package
Home

Objective

Utilize the debug package for selective debugging in multi-module programs and combine it with structured logging using winston.

Level 1: Single Module Debugging

Approach:

  • Install the debug package using npm install debug.
  • Set up a namespace for debugging within a single module. For example, use debug('app:module1').
  • Use the debug function to log messages within the module.

Testing:

  • Run the program with DEBUG=app:module1 node app.js to view debug messages.
  • Verify that debug messages only appear when the environment variable is set.

Level 2: Multi-Module Debugging

Approach:

  • Extend the program by creating multiple modules, each with its own debug namespace (e.g., app:module1, app:module2).
  • Ensure each module uses debug to log its specific messages.
  • Use the DEBUG environment variable to selectively enable debugging for specific modules.

Testing:

  • Enable debugging for specific modules by setting the DEBUG variable (e.g., DEBUG=app:module1,app:module2).
  • Confirm that only selected module debug messages appear in the output.

Level 3: Combining Debug and Winston

Approach:

  • Integrate winston for structured logging alongside debug. Install it using npm install winston.
  • Refactor the program to use winston for persistent logs and debug for transient debugging.
  • Enhance the compute program by making it interactive (e.g., using readline to accept user input).

Testing:

  • Run the program with DEBUG enabled and verify both transient debug logs and persistent winston logs.
  • Test the interactivity of the program and ensure logs provide meaningful insights.

Conclusion

By integrating debug and winston, this approach enables efficient selective debugging while maintaining structured logging. Modularization ensures scalability and better code organization.

Example debug setup in a module: 
const debug = require('debug')('app:module1');
const logger = require('winston');

// Debug example
debug('Debugging started in module1');

// Winston logging example
logger.info('Structured log: Module1 operation successful');
