Objective

The objective is to implement structured logging in a Node.js program using the winston library. The solution is divided into two levels, focusing on console logging and file transport with enhanced formatting.

Level 1: Basic Logging Setup

Approach:

  • Start by installing winston using npm install winston.
  • Create a logger instance with separate log levels: info, warn, and error.
  • Log essential program events like "Program started" (info level) and "Operation successful" (info level).
  • Add additional functions such as divide. For the division function, log division by zero as a fatal error.

Testing:

  • Run the program and verify logs appear in the console with appropriate levels and messages.
  • Ensure errors like division by zero are logged with severity.

Level 2: File Logging with Timestamps

Approach:

  • Enhance the logger by adding a file transport to log messages into a file (e.g., app.log).
  • Customize the log format to include timestamps for each log entry. Use winston.format.combine with timestamp and printf for a readable format.

Testing:

  • Run the program and confirm that logs are written to the specified file.
  • Check the file to ensure each log entry includes a timestamp and is formatted clearly.

Conclusion

By dividing the implementation into levels, the solution ensures a smooth transition from basic logging to advanced file transport with formatting. This approach makes the program's logging robust and suitable for debugging and monitoring.
