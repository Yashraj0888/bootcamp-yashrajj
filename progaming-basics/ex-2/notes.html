Objective

Write a Node.js program to accept and process command-line arguments, performing calculations and handling user inputs effectively. The solution is divided into three levels of increasing complexity.

Level 1: Basic Sum Calculation

Approach:

  • Create a script using Node.js that accepts numbers as command-line arguments.
  • Use process.argv to parse the arguments, ensuring the program can read inputs from the command line.
  • Implement error handling to check if the inputs are valid numbers. Display meaningful error messages if invalid inputs are provided.

Testing:

  • Run the script with valid numeric inputs and confirm the sum is correct.
  • Run the script with non-numeric inputs to ensure error handling is triggered.

Level 2: Adding Multiplication Support

Approach:

  • Enhance the script to support a --multiply flag. This flag modifies the operation to perform multiplication instead of addition.
  • Update the script to include a help text that provides usage instructions. Ensure clarity for the users.

Testing:

  • Test the --multiply flag with numeric inputs to confirm it calculates the product correctly.
  • Verify the help text displays usage instructions appropriately.

Level 3: Modularization and Global Command

Approach:

  • Create a new file, mathutils.js, to define the add and multiply functions. This ensures modularity and reusability of the code.
  • Refactor the main script to take the operation (e.g., add or multiply) as the first argument, followed by the operands.
  • Use the bin field in package.json to make the script executable as a global command. Utilize npm link to test this functionality.

Testing:

  • Run the program using the global command and validate the calculations for both addition and multiplication operations.
  • Test the modularized functions by importing them into another script.

Conclusion

The exercise is completed by building the script in incremental levels. Each step ensures clarity, functionality, and maintainability, while testing validates the program's robustness at every stage.

Example modularized function in mathutils.js: 
{
  "add": (a, b) => a + b,
  "multiply": (a, b) => a * b
}
