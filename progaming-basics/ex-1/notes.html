Objective

The objective is to configure and utilize npm for managing scripts, dependencies, and modules effectively. The solution is broken into three levels, each building upon the previous.

Level 1: Basic Setup

Approach:

  • Initialize the Node.js project using npm init, creating a package.json file for managing project configurations.
  • Add a start script to run the index.js file, enabling easy execution with npm start.
  • Include a dev script that uses DEBUG=* node index.js to enable detailed debugging outputs.

Testing:

  • Run npm start to ensure the basic program executes correctly.
  • Run npm run dev to verify debugging information is displayed as expected.

Level 2: Enhancing Functionality

Approach:

  • Install the chalk package and utilize it in index.js to style console outputs, improving clarity and readability.
  • Add nodemon as a development dependency to monitor and restart the application automatically during development.

Testing:

  • Run the program and check for styled console output using chalk.
  • Modify index.js and confirm that nodemon automatically restarts the program.

Level 3: Prestart Validation

Approach:

  • Define a prestart script in package.json to check if node_modules exists before starting the application.
  • In case node_modules is missing, include a command to reinstall dependencies using npm install during the prestart phase.

Testing:

  • Delete the node_modules folder and execute npm start. Confirm that dependencies are reinstalled and the program starts seamlessly.

Conclusion

Following a structured, incremental approach ensures a robust npm setup. Each level builds upon the previous one, with testing at every step to validate the configurations and functionality. This process ensures a reliable development workflow.

Example prestart script in package.json: 
{
  "scripts": {
    "prestart": "if [ ! -d \"node_modules\" ]; then npm install; fi",
    "start": "node index.js",
    "dev": "DEBUG=* nodemon index.js"
  }
}
