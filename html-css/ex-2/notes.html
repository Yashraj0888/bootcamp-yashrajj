CSS Techniques Used

The following CSS techniques were employed during the development of the webpage:

  • Font Styling: The body uses the Arial font family to maintain a simple, readable look. The header uses a larger font size for emphasis on the page title.
  • Text Alignment: The text in the header and footer is centered using text-align: center; to keep the design neat.
  • Spacing: Padding and margins are used throughout to ensure proper spacing between elements. The padding: 20px; rule is applied to the main content area, while 10px padding is applied to the footer.
  • Links and Hover Effects: Links are styled to have the primary color, and when hovered, the color changes to the secondary color. This provides clear visual feedback to the user.
  • List Styling: Unordered and ordered lists are used to organize the content. Items are spaced using the padding property to maintain readability.

Challenges Faced in CSS

Some of the challenges I encountered while working with CSS were:

  • Managing Layout Complexity: As the project involved multiple sections and varied layouts (floating navigation bars in Solution 2), managing spacing and positioning was challenging without using a CSS framework.
  • Inline CSS vs External CSS: While inline CSS worked well for smaller projects, I realized that as the project grew, it became harder to manage styles. I plan to move to external stylesheets in future projects for better maintainability.
  • Mobile Responsiveness: Ensuring the design worked well on mobile devices was a concern, especially with navigation bars that floated in the layout. I tackled this using media queries and testing the layout across multiple screen sizes.

Light and Dark Modes

For this project, I also implemented a light and dark mode toggle feature to enhance the user experience based on their preferences. This was achieved by utilizing CSS media queries and CSS variables:

  • Light Mode: In light mode, the background color is set to a light color (e.g., #f8f8f8), and the text color is a darker shade (e.g., #333333) to maintain readability in well-lit environments.
  • Dark Mode: For dark mode, the background color changes to a darker shade (e.g., #333333), and the text color is set to a lighter shade (e.g., #f8f8f8) to reduce eye strain in low-light environments.
  • Automatic Detection: The theme automatically switches based on the user's system settings using the @media (prefers-color-scheme) CSS media query. If the user prefers a dark theme, the page switches to dark mode without any additional actions.

By using CSS variables to define background and text colors for both light and dark modes, it becomes easy to switch themes dynamically without relying on JavaScript. The media query ensures the theme is responsive to user preferences, providing a seamless experience.
